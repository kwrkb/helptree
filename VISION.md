# VISION: helptree

## 🌍 The Problem (私たちが解決する課題)
現代のCLIツール（`docker`, `ffmpeg`, `gh`, `kubectl` など）は多機能化が進み、サブコマンドやオプションが膨大に存在します。
しかし、それらを理解するための標準的な手段は、依然として `コマンド --help` による **「巨大なテキストの壁」** をスクロールすることだけです。

開発者は必要な情報を探すために、ターミナルとブラウザを行き来し、認知負荷（Cognitive Load）を無駄に消費しています。「CLIツールの全体像を、直感的に把握できない」ことが最大のペインです。

## 💡 The Solution (解決策)
`helptree` は、複雑なCLIツールを解剖し、**インタラクティブなツリー構造として可視化するTUIビュアー**です。
線形（リニア）で読みにくいヘルプテキストを、階層的（ヒエラルキー）な地図へと再構築します。

## 🎯 Core Values (提供する3つの価値)
1. **全体像の瞬時な把握 (Bird's-eye View)**
   巨大なマニュアルを上から下へ読むのではなく、ディレクトリ構造を見るようにコマンドの全貌を俯瞰できます。
2. **未知の機能との遭遇 (Discoverability)**
   「こんな便利なサブコマンドがあったのか」という発見を、ツリーを展開する動作の中で自然に生み出します。
3. **ターミナル完結の流麗なUX (Seamless UX)**
   思考を分断することなく、キーボード操作のみでスムーズにコマンドの仕様を深掘りできます。
